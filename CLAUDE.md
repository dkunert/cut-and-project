## Project Overview

Mathematical research project investigating period lengths of one-dimensional cut-and-project sequences. The project has two implementations:
- **C** (`src/c/multiset/`): High-performance computation for testing mathematical conjectures
- **Python** (`src/python/`): Statistical analysis and ML modeling of computed results
- **LaTeX** (`LaTeX/`): Academic paper source and compiled PDF

## Build & Run Commands

### C Implementation

```bash
cd src/c/multiset
make all        # Build both performance (cnp) and debug (cnp_debug) versions
make perf       # Build optimized version only (-O3 -flto -funroll-loops -ffast-math)
make debug      # Build debug version only (-O0 -g)
make clean      # Remove binaries and object files
./cnp           # Run performance version
./cnp_debug     # Run debug version
```

### Python Implementation

```bash
cd src/python
make all        # Install dependencies in venv and run main.py
make install    # Create venv and install dependencies only
make run        # Run main.py (requires prior install)
make clean      # Remove virtual environment
```

## Configuration Before Running C Code

Edit `src/c/multiset/constants.h` before building/running:

- `X_MIN`, `X_MAX`: x-interval for computation (default: 0 to 1,000,000)
- `NUMBER_OF_CONJECTURE_TESTS`: Number of random test iterations (default: 1000)
- `TASKS`: Bitwise flags ŌĆö `TEST | TEST_CONJECTURES` (both enabled by default)
- `CREATE_FILE_TO_FIND_A_PATTERN`: Set to `true` to generate CSV output (default: `false`)
- `MAX_PERIOD_ARRAY_SIZE`: Controls memory allocation (~8GB at default of 8,000,000,000)

## Architecture

### C Code Structure

**`main.c`** ŌĆö Entry point: allocates aligned memory (`dx` array), then dispatches to `test()` and/or `test_conjectures()` based on `TASKS` flags.

**`mathematics.h` / `mathematics.c`** ŌĆö Core algorithm. Key types:
- `number_t` (`int_fast64_t`): Primary integer type
- `rational_t`: Struct with `numerator`/`denominator` fields

Key functions:
- `lambda(╬▒, ╬▓, ╬│, ╬┤, x_min, x_max, sort, dx)`: Core computation ŌĆö iterates x in [x_min, x_max], finds valid y integers, computes Euclidean distances, then finds period length
- `find_period_length()`: Brute-force period detection
- `dx_alloc()`: Allocates aligned memory for distance arrays
- Rational arithmetic: `rational_add`, `rational_subtract`, `rational_floor`, `rational_ceil`, `shorten`

**`test.c`** ŌĆö Unit tests for GCD, rational arithmetic, and period-finding logic.

**`conjectures.c`** ŌĆö Tests all 6 mathematical conjectures using random ╬▒, ╬▓ (coprime) and ╬│, ╬┤ (Žē ratio) values.

### Python Code Structure

**`main.py`** reads the CSV generated by the C code (`find_pattern_x_max_1000000_70977_lines.csv`) and runs four analyses:
1. XGBoost regression with grid search (27 hyperparameter combos, 3-fold CV)
2. Linear formula analysis via OLS (`statsmodels`)
3. Sinusoidal offset model fitting (`scipy.optimize.curve_fit`)
4. Quadratic offset model fitting

### Data Flow

C program ŌåÆ CSV file (`src/python/*.csv`) ŌåÆ Python analysis ŌåÆ statistical models and plots

The C program can generate the pattern CSV by setting `CREATE_FILE_TO_FIND_A_PATTERN true` in `constants.h`. The Python analysis reads this CSV.

## Mathematical Context

The project studies cut-and-project sequences defined by coprime ╬▒, ╬▓ Ōłł ŌäĢ, parameter Žē Ōēź 0, and an x-interval. Points are projected onto f(x) = (╬▒/╬▓)x, Euclidean distances are computed, and the period length ╬╗ is determined. The 6 conjectures relate ╬╗ to ╬ø_{╬▒,╬▓} (a reference period) depending on the value of Žē.
